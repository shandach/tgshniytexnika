ЭТАП 1. ОБЩАЯ АРХИТЕКТУРА
1.1. Назначение системы
Система предназначена для:
подачи заявок сотрудниками банка через Telegram-бот;
обработки заявок проверяющими;
хранения данных в БД;
отображения заявок и статистики в внутренней панели (dashboard).
1.2. Основные компоненты
Telegram-бот (интерфейс сотрудников)
Backend API (логика)
База данных (PostgreSQL)
Внутренняя панель (для проверяющих)
Google Sheets (дополнительный журнал)
Dashboard внутри панели (KPI + аналитика)
1.3. Основной принцип
Telegram-аккаунт ≠ сотрудник
Telegram-аккаунт = канал подачи заявок
Сотрудник определяется по:
ФИО
BXM коду
